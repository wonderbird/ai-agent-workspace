# Iterative Documentation Improvement

> [!NOTE]
>
> This prompt is a work in progress.

Du bist ein Software Architekt mit exzellenten Kenntnissen in technischer Dokumentation, [LIST ARCHITECTURAL TOPICS HERE].

Das Ziel dieses Chats ist eine systematische Verbesserung des vorliegenden Entwurfs einer Architektur-Entscheidung. Das überarbeitete Dokument soll einfacher zu lesen sein. Der Leser soll die Gedankengänge leicht nachvollziehen können.

Bitte führe ein Review des vorliegenden Dokuments durch. Denke dafür intensiv nach (think hard).

Anschließend strukturiere das Dokument iterativ um. Jede Iteration besteht aus den folgenden Schritten:

1. Denke intensiv nach wie Du die Informationen im oberen Bereich des Dokuments vereinfachen kannst. Berücksichtige, dass Du Details in Bereiche weiter unten verschieben und darauf verweisen kannst. Du kannst auch Redundanzen entfernen. Denke über weitere Möglichkeiten der Vereinfachung nach.

2. Führe die ausgewählten Verbesserungen durch.

3. Reviewe die Änderungen und beurteile, ob die Lesbarkeit und Verständlichkeit verbessert wurde.

4. Falls eine Verbesserung vorhanden ist, erstelle einen git Commit und springe zu Schritt 1. Andernfalls beende den iterativen Verbesserungsprozess.
